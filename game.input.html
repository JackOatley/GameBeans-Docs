GAME.input

The GAME.input object contains functions for dealing with user input such as the keyboard and mouse.

GAME.input.mouse

This is an object that contains several properties that are updated in the background by GameBeans that lets you determine the current state of the mouse, such as clicks and poisition. Please note that the mouse position is not updated if the mouse cursor is outside the game window.

Properties:

  • x - X position of the mouse within the window.
  • y - Y position of the mouse within the window.
  • pressed - List of buttons just pressed.
  • down - List of buttons being held down.
  • released - List of buttons just released.
  • wheelUp - Whether the wheel was just scrolled up, boolean.
  • wheelDown - Whether the wheel was just scrolled down, boolean.

Example

// print mouse position to screen GAME.draw.text( "mouse x = " + GAME.input.mouse.x, 320, 230 ); GAME.draw.text( "mouse y = " + GAME.input.mouse.y, 320, 250 );

https://developer.mozilla.org/en-US/docs/Web/API/MouseEvent/button

GAME.input.keyboard

This is an object that contains several properties that are updated in the background by GameBeans that lets you determine the current state of the keyboard keys. To check a certain key, supply the list relevant to the state you require, then supply the key code. For a reference of the different key codes see the link in the "Further" section below.

Properties:

  • pressed - List of keys just pressed.
  • down - List of keys being held down.
  • released - List of keys just released.

Example

// get input if ( GAME.input.keyboard.down["ArrowUp"] ) this.y -= 1; if ( GAME.input.keyboard.down["ArrowDown"] ) this.y += 1; if ( GAME.input.keyboard.down["ArrowLeft"] ) this.x += 1; if ( GAME.input.keyboard.down["ArrowRight"] ) this.x -= 1;

Further:

https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/code#Code_values

GAME.input.touch

This is an object that contains several properties that are updated in the background by GameBeans that lets you determine the current state of touches and poisitions. Each property of this object is an array, with the name being the state, and the index of the array being the index of the touch (or which finger in order pressed). Currently, only a single touch is handled, this is enough to mimic mouse support. Multi-touch support will be added in the future.

Properties:

  • x - X position of the touch within the window.
  • y - Y position of the touch within the window.
  • start - A new touch has started, similar to "mouse.pressed".
  • held - A touch continues to be held, similar to "mouse.down".
  • move - A touch has moved, internally this updates the touch position.
  • end - A touch has ended, similar to "mouse.released".
  • cancel - Currently the same as the "end" state.

Example

// print touch position to screen GAME.draw.text( "touch x = " + GAME.input.touch.x[0], 320, 230 ); GAME.draw.text( "touch y = " + GAME.input.touch.y[0], 320, 250 );

https://developer.mozilla.org/en-US/docs/Web/API/TouchEvent
