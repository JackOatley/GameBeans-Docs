GAME.main

The GAME.main object contains functions for dealing with the main game loop.

Contents:

  • GAME.main.setGameSpeed

GAME.main.setGameSpeed(speed)

Sets the speed of the game. It's important to know that this value determines how many times a second the Step Event should execute. The Draw Event will remain uneffected and always executes as fast as the browser allows, which is usually your monitor refresh rate. For example, if you have a 144hz monitor, your game will likely be executing the Draw Event 144 times a second. Where, by default, the step event always executes 60 times a second. This is why you shouldn't put game logic in the Draw Event. And you can use this function to determine the speed of your game logic. Also note, that any input will be saved until the next step, even if you set the game speed to be very slow.

Parameters:

  • speed - The speed you want the game to run at.
// Sloooooooowmoooooo... GAME.main.setGameSpeed(10);
