GAME.Tilemap

GAME.Tilemap is a class object, and so is used as a constructor. A tilemap is used to render a map made up of tiles from a single sheet/texture atlas (as a Sprite resource).

GAME.Tilemap( options )

Constructor

Creates a new tilemap.

Parameters:

  • options - OPTIONAL. You can use any, all or none of these parameters.
    • atlas - The texture atlas (a sprite) to use in this tilemap.
    • tileWidth - The width of each tile, in pixels, on the sheet.
    • tileHeight - The height of each tile, in pixels, on the sheet.
    • tileSpacing - Used if there's spacing between each tile on the sheet, in pixels.
    • tileOverlay - If there's an area around each tile that should overlap with other tiles, then this value sets that border in pixels.
    • tilesWide - How many tiles are there across the texture atlas. For example; N tiles wide.
    • mapWidth - How many tiles wide the Tilemap should be.
    • mapHeight - How many tiles high the Tilemap should be.

Example

// Setup tilemap global.tilemap = new GAME.Tilemap({ atlas: sprTiles, tileWidth: 16, tileHeight: 16, tileOverlay: 4, tilesWide: 6, mapWidth: global.map.width, mapHeight: global.map.height });

GAME.Tilemap.create( options )

Static

See Constructor for details.

Example

// Setup tilemap global.tilemap = GAME.Tilemap.create({ atlas: sprTiles, tileWidth: 16, tileHeight: 16, tileOverlay: 4, tilesWide: 6, mapWidth: global.map.width, mapHeight: global.map.height });

GAME.Tilemap.set( layer, x, y, index )

MethodStatic

Sets a cell in a Tilemap.

Parameters:

  • layer - Name of the player in which to place the tile.
  • x - The X grid position to place this tile.
  • y - The Y grid position to place this tile.
  • index - The tile index.

Example

// Set a tile, method global.tilemap.set("ground", 5, 5, 18); // Set a tile, static function GAME.Tilemap.set(global.tilemap, "ground", 5, 5, 18);

GAME.Tilemap.draw( options )

MethodStatic

Draw the Tilemap. An area of the Tilemap can be specified allowing easy selective rendering, though typically this will be the same as the camera viewing area.

Parameters:

  • options - OPTIONAL. You can use any, all or none of these parameters.
    • order - An array containing layer names in the order to render them.
    • left - The left hand side of the view box.
    • top - The top of the view box.
    • right - The right hand side of the view box.
    • bottom - The bottom of the view box.

Example

// Draw the tile map global.tilemap.draw({ order: ["water", "ground", "buildings"], left: global.camera.left, right: global.camera.right, top: global.camera.top, bottom: global.camera.bottom }); // Via static function GAME.Tilemap.draw(global.tilemap, sameOptionsAsAbove);
