Events

Events are a group of actions within an Object that execute at a particular time. Events can range from game loop moments like Step and Draw, to inputs and collisions between Objects.

Game

Create

The Create event is executed when a new instance of the Object is created.
In the case of an action such as Create Moving, the Create event is executed AFTER the changes to the speed and direction properties so the correct values will be present within the event.

Destroy

The Destroy event is executed when the instance is destroyed by the Destroy action. This can be useful for changing the score when you kill an enemy, or creating an explosion when a bomb is destroyed.

Step

Executed within the game logic loop, by default this should execute 60 times a second. There are 3 types of Step event; Normal, Begin and End.
The Begin Step event is executed before all other Events.
The Normal Step event is executed just before all instance properties are updated, such as; speed, gravity, animation, etc. And before other events like collisions and input.
The End Step event is executed just before the drawing events, but after all other events.

Draw

Used for drawing. Executed when a new frame is about to be rendered, as such this event can run a variable number of times a second so it should not be used for game logic! There are 2 types of Draw event; Normal and GUI.
The Normal Draw event is where' most drawing takes place, it's aware of any transformations such as cameras, and will handle correct drawing for you.
The GUI Draw event happens after the Normal Draw event and disables any transformations and scaling. As such if a sprite is drawn at 0, 0 then it will be drawn at 0, 0 in the window regardless of any cameras. This makes it perfect for drawing user interfaces.

Collision

Executed when the current instance collides with another instance of a given a Object. The keywords "all" and "solid" can also be used to select those groups of instances. An instance can NOT collide with itself, but it can collide with other instances of the same object as it.

Room Enter

The Room Enter event is executed for all instances that initially exist in a room or instances that are persistent between rooms, when a room is entered.

Room Leave

The Room Leave event is executed for all instances that exist before moving to another room.

Outside Room

The Outside Room event is executed when the instance lies completely outside the bounds of the room. This can be useful for deleting instances like bullets that leave the game area.

Input

These are events triggered directly my human interaction.

Keyboard

Executed when the given Keyboard key is in the given state. There is a wide selection of keys available. And three states for each; Press, Release and Hold.
Press events execute when the key is first pressed down.
Down events execute for each game step in which the key remains down.
Release events execute when the key stops being held down/is released.

Mouse

Executed when the given Mouse button is in the given state. The three main buttons are represented; Left, Right and Middle. All of which have three states which can be checked; Down, Press and Release.
There is also the WheelUp and WheelDown even types, which don't have states and just execute when the mouse wheel is turned.

Resize

Executed when the window/browser resizes.
